# AWS Cloud Infrastructure with Terraform 

Цей проєкт реалізує автоматизоване розгортання хмарної інфраструктури в AWS за допомогою Terraform. Використано модульний підхід для забезпечення гнучкості та повторного використання коду.


Структура проєкту

Проєкт розділений на логічні модулі для керування різними частинами інфраструктури:

* **`modules/s3-backend`**: Налаштування віддаленого сховища (S3 бакет та DynamoDB) для зберігання стану Terraform та запобігання конфліктам через механізм Locking.
* **`modules/vpc`**: Створення віртуальної приватної мережі (VPC), що включає 3 публічні та 3 приватні підмережі, налаштування Internet Gateway та таблиць маршрутизації.
* **`modules/ecr`**: Репозиторій для Docker-образів з активованим автоматичним скануванням на вразливості при завантаженні (scan on push).



Основні команди
Для роботи з проєктом використовуйте наступний цикл команд:

1.  **Ініціалізація** (завантаження провайдерів та налаштування бекенду):
    ```bash
    terraform init
    ```
2.  **Планування** (перегляд змін перед застосуванням):
    ```bash
    terraform plan
    ```
3.  **Розгортання** (створення ресурсів в AWS):
    ```bash
    terraform apply
    ```
4.  **Знищення** (повне видалення інфраструктури для економії коштів):
    ```bash
    terraform destroy
    ```

Конфігурація Backend

У проєкті налаштовано віддалений стейт-файл. Це дозволяє команді працювати над однією інфраструктурою без ризику пошкодити її стан.


## Розгортання та міграція Backend

Оскільки S3-бакет та DynamoDB для зберігання стейту створюються самим Terraform, процес розгортання відбувається у два етапи:

1. **Початкове створення інфраструктури**:
   * Тимчасово закоментуйте весь вміст файлу `backend.tf`.
   * Виконайте `terraform init` (буде використано локальний стейт).
   * Виконайте `terraform apply` для створення S3 та DynamoDB.

2. **Міграція стейту в хмару**:
   * Розкоментуйте вміст файлу `backend.tf`.
   * Виконайте `terraform init` знову.
   * На запит системи про копіювання локального стейту в S3 дайте відповідь `yes`.


> **Примітка:** Конфігурація знаходиться у файлі `backend.tf`. Перед першим запуском після повного видалення (`destroy`) блок backend має бути закоментований, доки не будуть створені ресурси S3 та DynamoDB.

* **Безпека**: Файли стану (`.tfstate`) та конфіденційні дані ігноруються системою Git завдяки налаштованому `.gitignore`.
